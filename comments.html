Overall Structure of my Website

The website is Structured into 4 html pages each focusing on differnet aspects of my experience and goals with computer science.

The pages interact with eachother through the navigation bar at the top of the page. The pages are styled using a css file that is linked to each page.

The Internal Structure of Each Web Page

  • Each page is structured with a header, main content and a footer.
  • The header contains the navigation bar and the logo of UTS.
  • The main content contains the main information of the page.
  • The footer contains the author of the page and the university.

Description of the Styles Used

Selector Type of Selector Style Description Usage
nav Element Selector Defines the navigation bars background color, padding, and alignment. Navigation Bar
.mainheader Class Selector Applies a large font size and centers the header text. Header on All Pages
.fade-in-content Class Selector Uses @keyframes to fade in the content on page load. Applied to main content
.flex-container-header Class Selector Flexbox layout for the header to center the title. Header Section
.past-content-container Class Selector Flexbox layout for content container on the "Past" page. Past Page Content
.img-container Class Selector Defines image and text layout for the TAFE image section. "Past" Page Image Section
table Element Selector Styles for table borders, text alignment, and spacing. Academic Records Table
footer Element Selector Fixed at the bottom of the page with a background color. Footer Section

Discussion of the graphic Design Aspects of the Website

Flexbox Layout: I used flexbox throughout the site for organizing content. Flexbox is highly responsive, which ensures that elements adjust smoothly based on screen size. This helps in creating a flexible and user-friendly design.

Two-Column Layout: On the "Past" page, I used a two-column layout, placing the text content on the left and the image on the right. This makes the content balanced and easy to read.

Tables: The academic records are presented in a clear and organized table, which allows users to easily view and compare the information.

Colors:

I used a black background for the navigation bar and footer to create contrast with the white text, improving readability.

The main content area uses a light background, creating a pleasant contrast with the darker elements, ensuring that the text is easy to read.

Typeface:

I used a standard web-safe font, Arial, for the text to ensure compatibility across different browsers and devices.

The headings (<h1>) have larger fonts to create emphasis and structure.

Discussion of the HTML alt Tag

The alt tag is used within the img tag to describe images for accessibility reasons. It provides a text description of the image for users who rely on screen readers, making the site more accessible to individuals with visual impairments.

How the site Will Work Without CSS

When CSS is disabled, the page structure will still be readable, but the layout will not have the same styling effects. For instance:

  • The navigation bar will no longer have a black background and white text; it will display the default browser styles.
  • The flexbox layout will be ignored, and the content will be stacked vertically.
  • The page will lose the visual appeal provided by CSS effects like hover animations, the fade-in effect, and the two-column layout.

Despite this, the pages content will still be visible, and the text will be readable, ensuring that the site remains functional even without CSS.
